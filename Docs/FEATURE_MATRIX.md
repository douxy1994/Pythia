# Pythia macOS / Windows Feature Matrix

| Feature | macOS Status | Windows Status | Notes |
| --- | --- | --- | --- |
| Main translation window | Implemented | Scaffolded | Windows Flutter shell has input, language selectors, translate, copy, clear, result cards, and history sidebar; Windows runtime verification still required. |
| Multi-service translation | Implemented | Implemented in code, runtime verification pending | Windows has Local, Google, Baidu, Youdao, OpenAI-compatible, DeepL, and LibreTranslate providers, Credential Manager-backed secrets, settings UI, and persisted multi-select service chips. Live Windows x64 verification is still required. |
| Language selection | Implemented | Scaffolded | Windows has initial language selectors using shared codes. |
| Smart mixed-language routing | Implemented | Scaffolded | Windows `TranslationServiceRegistry.resolvedLanguages` now mirrors the macOS mixed Chinese/English target-priority rule and feeds provider requests/history records; live Windows verification still required. |
| Settings persistence | Implemented | Scaffolded | Windows local JSON settings exist; platform-specific settings still need fuller UI. |
| API key storage | Local UserDefaults by user request | Native channel scaffold | Windows has Dart MethodChannel and C++ Credential Manager implementation registered by the included runner; live verification still required. |
| History save/search/delete | Implemented | Scaffolded | Windows can save/read/search/delete/clear visible history with logical deletion; live Windows verification still required. |
| History favorite/pin | Partial | Scaffolded | Windows favorite changes are persisted and marked pending sync; live Windows verification still required. |
| Local/WebDAV backup and restore | Implemented with shared schema | Implemented in code, runtime verification pending | Both use schema-v1 portable settings/history at `/Pythia/settings/portable-backup.json`, pre-backup and merge history, preserve device settings, and omit secrets. macOS retains legacy `/pot` restore fallback only. |
| WebDAV history sync | Implemented in code | Implemented in code, runtime verification pending | Both platforms support manual, startup, periodic, local-change-debounced, and best-effort exit sync. Automatic schedules persist an arbitrary positive value plus minute/hour/day/week unit and convert to exact seconds. Windows transient failures retry with bounded backoff, while auth/corrupt-data errors stop immediately. Live cross-device WebDAV verification remains required. |
| Global hotkeys | Implemented | Implemented in code, runtime verification pending | Windows records supported combinations directly in settings, canonicalizes aliases/order, rejects malformed or duplicate actions, registers show-window/selection/screenshot actions through `RegisterHotKey`, and dispatches `WM_HOTKEY` to Dart. Live conflict testing on Windows remains required. |
| Selection translation | Implemented | Implemented in code, runtime verification pending | Windows reads the focused control's selected TextPattern ranges through UI Automation without touching the clipboard, then falls back to simulated copy only when UIA is unavailable. Clipboard sequence checks prevent stale clipboard text from being mistaken for a selection. |
| Screenshot OCR/translation | Implemented with macOS Vision | Implemented in code, runtime verification pending | Windows hides Pythia, freezes the virtual desktop across monitors, supports drag selection/cancel, captures selected pixels, recognizes text with Windows Runtime `OcrEngine`, restores the window, and feeds the text into translation. Windows SDK CI and live language-pack verification remain required. |
| Status bar / tray | Implemented | Native scaffold complete, live verification pending | Windows runner installs a notification-area icon; left-click restores the window; the right-click menu supports show, quick input translation, settings, history, WebDAV history sync, and quit. Business actions are routed back to tested Dart handlers. |
| System notifications | Native status feedback | Implemented in code, runtime verification pending | Windows startup/periodic WebDAV sync uses native information/error notifications and a persisted user switch; manual sync does not duplicate in-window feedback. |
| Window behavior | Mostly implemented | Native scaffold complete, live verification pending | Both General settings pages expose real startup toggles. macOS applies `SMAppService.mainApp` and reports approval/errors; Windows writes/removes the current-user Run entry. Windows also persists close-to-tray, always-on-top, hide-on-blur, and window placement. |
| Update check | Implemented against Pythia GitHub | Implemented in code, signed runtime verification pending | Windows requires an exact Pythia Windows x64 EXE/MSIX plus `.sha256`, restricts downloads and redirects to GitHub HTTPS, verifies size/SHA-256, asks for confirmation, requires valid Authenticode through `WinVerifyTrust`, launches the installer, and exits. |
| Release architecture/package gate | Implemented for macOS package script | x64 verifier scaffolded | Windows CMake rejects non-x64 toolchains. `tool/verify_release_package.dart` requires AMD64 PE machine `0x8664` and rejects `.potext`, legacy plugin runner/source trees, and secret-like payloads. Must be run against the real Windows release directory after `flutter build windows --release`. |

This matrix is intentionally conservative. A feature is not marked complete unless there is current code and verification evidence.
